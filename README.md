# brain.md

The **Open Project Brain Standard** — a neutral specification for *how project knowledge is captured as plain files* — together with a **Brain plugin** that runs standalone, independent of any client.

It lets coding agents such as Claude Code / Codex learn how to work with the `brain/` directory simply by reading [`BRAIN.md`](./BRAIN.md) at the repository root — **drop it in and it just works, with zero runtime dependencies** (v0.1 needs no service and no MCP server).

## What this is

Project knowledge shouldn't live only inside some SaaS. brain.md captures it as plain Markdown in the repository:

- Project-level deliverables → 6 **root pages** under `brain/` (background / architecture / flow / mindmap / stack / roadmap).
- Incremental knowledge → individual **pages** under `brain/pages/` (five categories: decision / concept / project / person / reference), each = a rewritable `compiled_truth` + an append-only `timeline`.
- The AI's operating manual → **skills** under `.claude/skills/` (`page-management`, `ingest`).

A coding agent understands these conventions just by reading `BRAIN.md`; deterministic scripts handle validation and index rebuilding.

## Structure of the three-piece set

```
BRAIN.md                      # protocol entry point: the AI reads it to learn how to work with brain/
AGENTS.md / CLAUDE.md         # each @import ./BRAIN.md and pick up the skill rules
brain/
  background.md  architecture.md  flow.md            # root pages (the fixed six —
  mindmap.md     stack.md         roadmap.md          #   only updated, no timeline)
  index.md                                            # generated by reindex
  pages/<id>.md                                       # incremental knowledge pages
.claude/skills/
  page-management/SKILL.md     # page category taxonomy + write protocol
  ingest/SKILL.md              # digest input and write it down as brain content
scripts/
  reindex.mjs      # scan brain/pages/ → rebuild brain/index.md
  lint-links.mjs   # check for broken [[page-id]] wiki-links
  validate.mjs     # check frontmatter schema + back-stop missing timeline entries
```

## How to use it

1. Copy this repository's `BRAIN.md`, `brain/`, `.claude/skills/`, and `scripts/` into your project (or start from this repository directly).
2. Add a line `@import ./BRAIN.md` to your project's `AGENTS.md` and `CLAUDE.md`.
3. Let the coding agent work as usual — it will read and write `brain/` following the conventions in `BRAIN.md`.
4. Run the checks before committing:

   ```bash
   node scripts/reindex.mjs      # rebuild the index
   node scripts/lint-links.mjs   # check for broken wiki-links
   node scripts/validate.mjs     # check schema + timeline back-stop
   # or all at once:
   npm run check
   ```

   The scripts are pure Node.js with **zero npm dependencies** — no `npm install` required.
